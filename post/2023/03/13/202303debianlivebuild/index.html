Debian Live Buildに関するメモ

2023-03-13
Share
  Tweet  

はじめに

日本国内における個人や学生開発コミュニティでLinuxディストリビューション開発が盛んになっている．これらのディストリビューションは，DebianやUbuntuなどのメジャーディストリビューションをベースにし，パッケージリポジトリもメジャーディストリビューションのものを流用しているものが大半を占めており，実態はメジャーディストリビューションのリマスタ版である．これらのリマスタ版の開発では，Live Imageによるインストール形式が採用されている．そこで，私自身もオレオレ仕様のDebianリマスタ版を開発するべく，まずLive Imageの作成方法について調査した．

目次
  • はじめに
  • Live Build環境の導入
  • Live Build プロジェクトの作成

Live Build環境の導入

apt install live-build

Live Build プロジェクトの作成

lb config \
	--apt apt \
	--architecture amd64 \
	--distribution bookworm \
	--distribution-chroot bookworm \
	--distribution-binary bookworm \
	--archive-areas "main contrib non-free non-free-firmware" \
	--mirror-bootstrap "http://ftp.jp.debian.org/debian" \
	--mirror-chroot "http://ftp.jp.debian.org/debian" \
	--mirror-binary "http://ftp.jp.debian.org/debian" \
	--bootappend-live "boot=live components debug=1" \
	--image-name "test"

lb build
